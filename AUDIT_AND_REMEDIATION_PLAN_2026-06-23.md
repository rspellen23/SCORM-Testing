# Course Builder ‚Äî Audit & Remediation Plan (2026-06-23)

Locked plan from the full-system engineering audit (5 parallel specialist passes:
security, parser/registry, SCORM/cmi5 conformance, player/accessibility,
architecture/testing). **Next session executes this top-to-bottom.** Check items off
as completed. Each item has severity, `file:line` evidence, the fix, and a
definition-of-done.

## Root cause (why these exist)
Three reinforcing gaps make correctness errors **silent** and drift **invisible**:
1. The system is engineered to **never crash** ‚Üí malformed input is dropped to `""`/empty, not flagged. The lint is the only safety net and checks only 2 things.
2. **No central block registry** ‚Üí adding one block type means editing ~9 scattered sites that nothing cross-checks ‚Üí they drift.
3. **No tests / no CI / not a git repo / not a package** ‚Üí a parser regression makes a structurally-valid-but-wrong course that `scorm_lint` passes clean.

> The `imageText` merge bug fixed 2026-06-23 (denylist‚Üíallowlist) and the still-live
> PPTX quote/infographic drop are the SAME disease. Fixing the root cause (registry +
> exhaustive dispatch + tests) retires the whole class.

## What NOT to "fix" (verified good ‚Äî leave alone)
- Runtime stdlib-only core is real: the SCORM build path imports zero third-party libs; `python-pptx`/`docx`/`PIL` confined to export surfaces. Air-gapped builds work with bare `python3`.
- Brand externalization is clean (`grep -ri teletracking src/` empty).
- **cmi5.xml validates clean against the official ADL `CourseStructure.xsd`.** SCORM 1.2 manifest matches the canonical RTE example.
- `scorm_lint` deliberately not running `xmllint` (documented reason `scorm_lint.py:110-115`) is a correct senior call ‚Äî libxml2 false-fails the conformant manifest. Confirmed reproducible.
- Reduced-motion handling for the new entrance animations is thorough and correct.
- The **chart block accessibility** (`chart_svg.py`: `role="img"` + sr-only data table) is the model to copy to the other interactive blocks.

## PHASE 5 ‚Äî Security (local tool; lower real risk but real)

- [x] **5.1 No CSRF/Origin check (HIGH-in-context).** Any webpage the user visits can drive the localhost API (body parsed as JSON regardless of content-type). Fix: validate `Origin`/`Referer` against `127.0.0.1:<port>` on every POST + per-session CSRF token + require `application/json`. (`server.py:572-613`) ‚Äî **DONE 2026-06-23.** `do_POST` now calls `_csrf_ok()` first ‚Üí 403 unless ALL hold: Host hostname ‚àà {127.0.0.1, localhost} (blocks DNS-rebinding), `Origin`/`Referer` == this server's own origin, `Content-Type: application/json`, and `X-CSRF-Token` == a per-process `secrets.token_urlsafe` (compared with `compare_digest`). The token is stamped into the served `index.html` (`_serve_index` replaces `__CSRF_TOKEN__`); the page reads it from a `<meta>` and a one-time `window.fetch` wrapper attaches the header + JSON content-type to every POST (no call-site churn). Live-tested: no-token, evil-Origin, non-JSON, and spoofed-Host POSTs all 403; the legit token+origin+JSON POST passes.
- [x] **5.2 Arbitrary file read (HIGH-in-context).** `/api/readjson`, `/api/ls`, `/preview` (`startswith` guard) are unconfined. Fix: `realpath` + `os.path.commonpath([root,target])==root` against an allowlist of roots. (`server.py:546-568`) ‚Äî **DONE 2026-06-23.** `_within_roots()` resolves symlinks then checks `commonpath` against an allowlist (`~`, install ROOT, tempdir, `/Volumes`) ‚Äî broad enough to keep the navigator useful, blocks `/etc` + other users' homes. `/api/readjson`/`/api/scan`/`/api/listfiles` reject out-of-root; `/api/ls` clamps out-of-root back to `~`. `/preview` rewritten: `realpath` both, require base within roots AND `commonpath([base,target])==base` (kills the `startswith` prefix bug + traversal). Live-tested: `/etc/hosts` readjson blocked, allowed-root JSON reads, `/preview?dir=/etc` 404s, `ls /etc` clamps to home.
- [x] **5.3 Build argv hardening + `/api/reveal` (MED).** Prepend `--` before positionals, reject `-`-leading path args, validate `brand` against `list_brands()`; confine `/api/reveal` to non-executable workspace paths. (`server.py:239-275,481-492`) ‚Äî **DONE 2026-06-23.** `_safe_path_arg()` rejects `-`-leading `md`/`images`/`out` (build) so argparse can't misread a path as a flag; `_safe_brand()` confines `--brand` to `list_brands()` in both `build_jobs` and `do_deck`; deck `transition`/`transition_dir` validated against fixed sets. `/api/reveal` now requires same-origin (Referer/Origin) AND `_within_roots(path)` before shelling `open`.
